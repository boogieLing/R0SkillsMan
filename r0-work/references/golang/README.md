# Golang Engineering References for r0-work

This directory defines **mandatory engineering constraints** for Golang projects
executed under the `r0-work` skill.

## Applicability

These rules MUST be applied when ALL conditions are met:

- Project language is Golang
- Task category is Backend or Backend-related infrastructure
- Code is production-facing or library-level

## Authority

All documents in this directory are considered **hard constraints**.

Violating any rule defined here is considered a **skill violation**, even if
the main skill document does not restate the rule explicitly.

## Covered Domains

- Error handling semantics
- Logging coverage and observability
- Panic usage policy
- Code style and structure
- Additional best engineering practices
