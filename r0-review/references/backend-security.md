# Backend Security Checklist

适用场景：后端服务、API、任务处理系统。

## 身份与权限
- 每个敏感操作都应有鉴权与权限校验。
- 多租户场景必须验证租户隔离条件。
- 默认拒绝策略是否生效（未命中规则即拒绝）。

## 输入与输出
- 所有外部输入都应做类型、长度、格式、范围校验。
- 响应与日志不得泄露密钥、隐私字段、内部拓扑。
- 错误信息对外最小化，对内保留可追踪上下文。

## 注入与路径攻击
- SQL 查询是否参数化，禁止拼接构造。
- 命令执行是否可控，避免 shell 注入。
- 文件路径是否做规范化与目录边界检查，防止路径遍历。
- 外部 URL 请求是否防 SSRF（协议/网段/域名白名单）。

## 鲁棒性
- 超时、重试、幂等、熔断是否完备。
- 并发共享状态是否有锁或无锁正确性保证。
- 异常分支是否释放资源并写出可定位日志。
